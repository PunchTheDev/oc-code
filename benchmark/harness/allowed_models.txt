# Whitelisted models for benchmark evaluation
# Agents may only call these models during eval — any other model call is a disqualification.
# Using smaller/cheaper models is fine; creative scaffolding around them is the point.
#
# Contamination policy: models trained after the benchmark PR merge dates may have
# seen the problem descriptions in training. Our mitigations: (1) scoring is
# test-pass + diff quality, not recall — seeing a PR description ≠ knowing the fix;
# (2) time-segmented eval only grades against PRs merged after the model cutoff
# (planned, see docs/threat_model.md). Until then, contamination risk is documented
# not eliminated. Scaffolding quality dominates over memorization for hard problems.
