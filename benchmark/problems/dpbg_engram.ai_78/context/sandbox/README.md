# Sandbox Container

Isolated execution environment for running untrusted code generated by the Meta-Programmer.

## Purpose

The sandbox provides a secure, resource-limited container where generated code can be tested without risk to the host system or other containers.

## Security Features

- **Network Isolation**: Containers are spawned with `network_disabled=True`
- **Read-Only Filesystem**: Set with `read_only=True` (except /tmp)
- **Resource Limits**:
  - Memory: 512MB default (configurable via `SANDBOX_MEMORY_LIMIT`)
  - CPU: 50% of one core (configurable via `SANDBOX_CPU_QUOTA`)
  - PIDs: 100 max processes (configurable via `SANDBOX_MAX_PIDS`)
  - Timeout: 30 seconds (configurable via `SANDBOX_TIMEOUT_SECONDS`)
- **Non-root User**: Runs as `sandbox` user (UID 1000)
- **Auto-destroy**: Containers are automatically removed after execution

## Usage

The sandbox is not run directly. Instead, it's spawned on-demand by the Meta-Programmer:

```python
# Meta-Programmer spawns sandbox for testing
container = docker_client.containers.run(
    image="activelearning-sandbox:latest",
    command=["pytest", "/sandbox/test_generated.py"],
    volumes={
        "/path/to/code": {"bind": "/sandbox", "mode": "ro"}
    },
    network_disabled=True,
    read_only=True,
    mem_limit="512m",
    cpu_quota=50000,
    pids_limit=100,
    detach=True,
    remove=True  # Auto-destroy after completion
)
```

## Performance Targets

- **Unit tests**: ~5 seconds total runtime
- **Startup overhead**: <1 second
- **Result collection**: <1 second

## Building

The sandbox base image is built automatically via docker-compose:

```bash
docker compose --profile build-only build sandbox-base
```

Or manually:

```bash
docker build -t activelearning-sandbox:latest sandbox/
```

## Testing the Sandbox

Test the sandbox isolation:

```bash
# Test basic execution
docker run --rm activelearning-sandbox:latest python -c "print('Hello from sandbox')"

# Test network isolation
docker run --rm --network none activelearning-sandbox:latest \
    python -c "import socket; socket.create_connection(('google.com', 80))"
# Should fail: socket.gaierror

# Test pytest
docker run --rm activelearning-sandbox:latest pytest --version
```

## Limitations

- No network access (by design)
- No GPU access (not needed for unit tests)
- Limited to Python 3.11 standard library + pytest
- Cannot install additional packages at runtime

For integration tests with hardware mocks and NATS, use the `test-runner` container instead.
